# Phase 2 roadmap

## 1. GitHub Checks API support

Add richer GitHub Checks summaries in addition to the PR comment.

## 2. Optional PR labels

Support configurable labels such as `risk/low`, `risk/medium`, `risk/high`, `needs-tests`, and `large-change`.

## 3. SARIF output

Generate SARIF for TODO/FIXME and quality findings to improve code scanning integration.

## 4. HTML report artifacts

Publish optional HTML reports for teams that want a shareable summary outside the PR thread.

## 5. Monorepo support

Support package-level configuration for larger repositories.

## 6. Coverage regression mode

Compare coverage against the base branch instead of relying on a single threshold.

## 7. Language presets

Add presets for Node.js, React, Python, Java Spring, and Terraform workflows.

## 8. Config schema documentation

Document a JSON schema for `.github/pr-quality-bot.yml` to improve editor support.

## 9. GitHub Marketplace readiness

Prepare branding, screenshots, and a support policy for Marketplace publication.

## 10. Optional AI summary mode

Keep AI support disabled by default and never require it for the core workflow.
